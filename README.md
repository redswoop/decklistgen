# DecklistGen

A Pokemon TCG proxy tool. Browse cards, build decklists, generate cleaned artwork, and print proxy cards.

## What It Does

- **Browse** every card in the Scarlet & Violet and Sword & Shield eras (data from TCGdex API)
- **Filter** by set, rarity, energy type, category, special attributes (ex, V, VMAX, etc.), full-art, foil
- **Build decklists** by clicking cards, export as text or download
- **Generate cleaned artwork** — uses AI (Flux Klein 9B via ComfyUI) to remove text from card art
- **View proxy SVGs** generated by the built-in pokeproxy renderer

### Components

**DecklistGen** — the main application and data owner.
- Bun server with Hono routes, Vue 3 client with TanStack Query
- Fetches card data from TCGdex, caches as JSON in `cache/`
- Serves the browsing/filtering/decklist UI
- Calls ComfyUI directly for image cleaning (no framehouse dependency)
- Calls pokeproxy subprocess for SVG proxy generation

**Pokeproxy** (`pokeproxy/`) — stateless SVG renderer, lives inside this repo.
- `pokeproxy.py` — SVG layout engine (~1000 lines)
- `bridge.py` — stdin/stdout JSON bridge for subprocess calls
- Pure function: receives card data + image bytes, returns SVG string
- No fetching, no caching, no state — decklistgen owns all data

**ComfyUI** — external GPU server running Flux Klein 9B for image cleaning.
- Called directly via `src/server/services/comfyui.ts`
- Not a runtime dependency — everything works without it except on-demand cleaning

### Data Flow

```
Card browsing:
  TCGdex API → server caches JSON → card-store (in-memory) → API → Vue client

SVG proxy generation:
  Server sends card JSON + image base64 to pokeproxy/bridge.py via stdin
  → bridge.py calls generate_svg() or generate_fullart_svg()
  → SVG returned on stdout → cached in cache/{cardId}.svg

Image cleaning (optional):
  User clicks "Generate" → server sends image to ComfyUI directly
  → Klein 9B generates text-free version → saved as cache/{cardId}_clean.png
  → Composite (original top + cleaned bottom) saved as cache/{cardId}_composite.png

Decklist export:
  User adds cards via UI → DecklistPanel tracks entries → Export as text file
```

### Cache Structure

Single flat directory. All artifacts for a card share the same `{setId}-{cardNum}` prefix:

```
cache/
  sv06.5-1.json              ← card metadata (from TCGdex API)
  sv06.5-1.png               ← original card image (600x825, from TCGdex)
  sv06.5-1_clean.png         ← AI-cleaned image (736x1024, from ComfyUI)
  sv06.5-1_composite.png     ← top 20% original + bottom 80% cleaned
  sv06.5-1.svg               ← proxy card SVG (from pokeproxy renderer)
```

## Running

### Prerequisites
- [Bun](https://bun.sh) runtime
- Python 3 with venv (for pokeproxy SVG renderer)
- For image cleaning: ComfyUI with Klein 9B model

### Setup
```bash
bun install

# Set up pokeproxy Python environment
cd pokeproxy && python3 -m venv .venv && .venv/bin/pip install -r requirements.txt && cd ..
```

### Development
```bash
bun run dev          # starts server (3001) + client (5173) concurrently
```

### Production
```bash
bun run build        # builds client + server
bun run start        # serves on port 3001
```

### CLI
```bash
bun run cli -- --set SFA --category Pokemon --fullart
bun run cli -- --list-filters
bun run cli -- --set SFA --output filtered.txt
```

### Environment Variables
| Variable | Default | Description |
|---|---|---|
| `PORT` | `3001` | Server port |
| `COMFYUI_URL` | `http://stormer.local:8188` | ComfyUI server URL (for image cleaning) |
